# Windows packaging

Release builds for Windows produce three artifacts in `dist/`:

| Artifact | Format | Notes |
|----------|--------|--------|
| `kaiting-x.y.z-windows.zip` | Portable | Unzip and run `kaiting.exe` |
| `kaiting-x.y.z-windows.msix` | MSIX | Modern installer; self-signed for sideload |
| `kaiting-x.y.z-windows-setup.exe` | Inno Setup | Classic wizard installer |

## Build

From the repo root on a Windows host:

```powershell
dart run tool/release.dart windows
# or keep the current version:
dart run tool/release.dart windows --no-bump
```

## Prerequisites

1. **Flutter Windows desktop** toolchain (Visual Studio with C++ desktop workload).
2. **MSIX** — provided by the `msix` dev dependency (`flutter pub get`).
3. **Inno Setup 6** — required for `*-setup.exe`:

   ```powershell
   winget install --id JRSoftware.InnoSetup -e --accept-package-agreements
   ```

   Or install from https://jrsoftware.org/isinfo.php  
   The release script looks for `ISCC.exe` under the usual Program Files paths.

## Sideloading MSIX

The release MSIX is signed with a **test certificate** generated by the `msix`
package (or your own cert if configured). First-time install may require:

- Enabling sideloading: *Settings → Privacy & security → For developers → Developer Mode*  
  or *Install apps from any source*
- Trusting the certificate if Windows SmartScreen prompts

For Microsoft Store submission, set `store: true` in `msix_config` and fill in
Partner Center identity values.
