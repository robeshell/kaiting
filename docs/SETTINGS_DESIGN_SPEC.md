# 开听 设置系统设计规范

本规范覆盖设置首页、行内展开区域、底部选项面板、设置内页以及来源扫描、下载、清理
等流程型能力。它是 `DESIGN_FOUNDATION.md` 在设置场景中的强制细化规范。

## 1. 设计目标

- 设置页是一张安静、连续的表单，不是组件陈列页或第二套应用导航。
- 通过对齐、留白和细分隔线表达层级，避免依赖彩色图标、独立卡片和阴影。
- 相同能力在不同平台共享状态与语义，不强求共享完全相同的容器结构。
- 一项设置只能有一个主要完成路径；保存时机、取消结果和错误反馈必须可预期。

## 2. 信息架构

桌面和宽平板使用：

1. 播放、资料库、外观、操作、关于横向分类锚点；
2. 最大宽度 920 px 的单列表单内容；
3. 音乐来源、离线与缓存、问题与诊断作为独立内页。

设置首页不重复显示“设置”大标题；当前位置已经由应用主导航表达。手机和窄分屏不展示
横向分类锚点，直接使用分组列表。桌面应用主侧栏已经
提供一级导航，设置页不得再增加第二条竖向侧栏。

## 3. 栅格、尺寸与文字

- 内容最大宽度：920 px；页面边距沿用 compact / medium / wide 的 16 / 24 / 32 px。
- 桌面设置行最小高度 64 px；紧凑设置行最小高度 58 px。
- 大分组间距 28 px；行间使用 1 px hairline，不使用卡片间隙。
- 桌面分类栏与标题、正文共用左侧轴线，高度 48 px；固定时仅使用底部 hairline，不另设色块。
- 独立设置内页标题使用全局 26 / 28 px 标题 token；设置首页不显示重复页标题。
- 分组标签使用 12.5 px / 600 和 muted text，只负责标记边界，不与设置项标题竞争。
- 设置名称：桌面 14.5 px、移动端 14 px / 600；解释文字 12–12.5 px。
- 桌面名称与解释优先位于同一文字流；窄屏自动改为上下两行。
- 当前值、开关和操作统一靠右；同组控件必须共享右侧基线。

## 4. 颜色和材质

- 页面、内页和展开区只能读取 `ColorScheme`、`SoundGlassTheme` 与设置语义 token。
- 标题使用 primary text，说明与当前值使用 secondary text，短元数据才使用 muted text。
- 普通行透明且无投影；hover / focus / pressed 使用主题生成的弱中性覆盖色。
- 展开区统一使用 `surfaceContainerLow` 派生的实色表面，并以顶部、底部 hairline 收口；
  不为每种选择器发明不同背景。
- 强调色仅用于选中、当前值、焦点和主操作。来源类型色只用于真实来源或状态标识。
- 错误、警告和成功必须使用 `ColorScheme.error / tertiary` 或既有语义状态色，禁止页面
  直接使用 `redAccent`、`orangeAccent` 等原始颜色。

## 5. 设置行类型

### 即时设置

开关、单选和播放模式选择后立即生效。桌面可在统一行内面板展开；手机使用底部面板，
选择后关闭。当前值始终显示在设置行右侧。

### 分阶段编辑

需要多个参数的设置使用“取消 / 使用”流程，例如自定义主题色。编辑过程在局部预览，
只有“使用”才写入应用状态；取消、系统返回和点击遮罩不得改变原值。

### 导航型设置

音乐来源、离线与缓存、问题与诊断进入独立内页。内页统一使用返回、标题、说明、页面
操作和 920 px 内容宽度；系统返回与页面返回必须等价。

### 危险设置

删除来源、清空缓存等不可直接由普通行单击完成。必须在 `SoundDialog` 中说明影响范围，
使用语义错误色，并明确区分取消和执行。

### 流程型能力

扫描、连接、下载、重试和清理必须呈现 idle / running / success / warning / error / cancelled
状态。进行中操作应提供取消；失败应保留原因和可行的恢复动作；完成或失败通过行内状态
优先表达，SnackBar 只作为一次性结果确认。重复点击不得启动并发的同一任务。

## 6. 展开区

- 同一时间允许多个外观预览区保持展开，以支持连续比较。
- 展开内容不能改变所在设置行的左右基线。
- 皮肤与播放器样式使用响应式预览网格；主题色使用紧凑色板。
- 选中状态由强调色边界、勾选和可访问语义共同表达，不能只依赖颜色。
- 展开和收起使用 160–220 ms 的弱位移动画；系统减少动态效果时直接切换。

## 7. 响应式规则

| 场景 | 导航 | 选择器 | 内容布局 |
| --- | --- | --- | --- |
| 手机/封面屏 | 分组列表 | 底部面板或纵向展开 | 单列，16 px 边距 |
| 折叠屏展开/窄平板 | 可滚动横向分类或分组列表 | 2 列预览 | 避开中轴，24 px 边距 |
| 宽平板 | 横向分类 | 2–3 列预览 | 最大 920 px |
| 桌面 | 置顶横向分类 | 3–4 列预览 | 最大 920 px，32 px 边距 |

当窗口高度不足 600 px 时，优先保证内容滚动、操作可达和弹窗键盘避让，不压缩文字或
触摸目标。

## 8. 可访问性与验收

- 所有设置行、选项、展开状态和流程状态提供 Semantics。
- 键盘焦点顺序遵循视觉顺序；Enter / Space 激活，Escape 关闭面板或返回。
- 点击目标不小于 44×44 px；正文与背景满足 WCAG AA 对比度。
- 每种设置类型至少包含默认、选中、禁用、错误和恢复路径测试。
- 设置首页需覆盖 390、600、834、1100 px 宽度；内页需覆盖短桌面窗口和系统文字缩放。
