# Sound Next 开发看板

最后更新：2026-07-13

本文件是产品范围和开发顺序的唯一事实来源。规划分为多个能力域；每个能力域
最多只能有一张进行中的卡，修改共享播放或资料库核心时仍需串行。代码完成后
移入**待验证**；只有验收条件和相关平台构建全部通过，才能移入**已完成**。

## 产品能力地图

平台适配只是其中一条工作流。第一版需要同时推进播放器、资料库、远程来源、
性能稳定性和发布工程；不能以“某个平台能构建”代替播放器产品能力完成。

| 能力域 | 已有基础 | 第一版必须补齐 | 后续增强 |
| --- | --- | --- | --- |
| 平台适配 | Flutter 多端工程；macOS/Android 实播；Android/iOS 后台媒体适配层。 | iPhone/iPad Files 与分屏；Android/iOS 真机；Windows 本地/WebDAV/SMTC；音频焦点与中断恢复。 | 桌面托盘、系统快捷键、拖放、平台安装与自动更新。 |
| 播放器核心 | 播放/暂停/seek/切歌/自动下一首；真实进度源；队列与会话恢复。 | 顺序/单曲循环/列表循环/随机；队列查看、调整、删除、下一首播放；无缝切歌；完整格式矩阵。 | 淡入淡出、ReplayGain/音量标准化、均衡器、倍速和输出设备选择。 |
| 资料库与组织 | SQLite 资料库；本地与 WebDAV 扫描；专辑/歌曲/歌词/封面；原子重扫。 | 搜索、筛选、排序；艺人/专辑/流派入口；多碟专辑；增量扫描；收藏、历史和播放列表。 | 智能播放列表、元数据编辑、重复项与移动文件识别、统计。 |
| 网络与离线 | WebDAV 连接、凭据、目录扫描、Range 播放、seek、LRU 缓存。 | 离线下载管理；网络切换与断线重试；缓存状态、用量和清理 UI；来源失效恢复。 | 多来源合并、冲突处理、可选的其他音乐服务器协议。 |
| 性能与资源 | 元数据在 isolate 解析；SQLite 持久化；基础图片与音频缓存。 | 1,000/10,000 首基准；分页查询和列表虚拟化；启动、扫描、搜索、内存、图片和电量优化。 | 超大库压力测试、后台扫描调度和性能回归基线。 |
| 稳定性与数据安全 | 扫描事务、旧播放事件隔离、缓存并发保护、权限状态模型。 | 可重试错误；离线/凭据过期/损坏音频诊断；数据库迁移、备份恢复和异常退出恢复。 | 日志导出、崩溃诊断和可选的隐私友好遥测。 |
| 产品体验 | 原设计语言；响应式资料库、专辑页、正在播放页和迷你播放器。 | 空状态与首次引导；同步歌词体验；键盘/手势；无障碍、动态字体与中英文。 | 主题定制、动效、桌面紧凑模式和更多视觉细节。 |
| 工程与发布 | 自动化测试、静态检查和各平台 Debug 构建流程。 | CI 构建矩阵；集成/截图回归；版本迁移测试；签名、安装包、隐私说明和开源许可。 | Beta 分发、崩溃符号管理、发布自动化和升级策略。 |

## 当前焦点

当前实际实施仍是 **SND-405 — iPhone/iPad 自适应布局与 Files 回归**。
它完成后不再沿着平台清单单线推进，而是从三条主线各取一张卡：

- 播放器：SND-203 播放模式与可编辑队列。
- 产品资料库：SND-501 搜索、筛选与排序。
- 性能：SND-502 先建立基准，再让后续功能带着指标开发。

Windows 媒体控制受当前主机限制，SND-401/SND-403 保持阻塞，不妨碍
播放器和资料库功能继续开发。

队列中的卡不是一条必须完全串行的长链。平台、播放器和性能三条主线可以并行，
但同一时间不要让多个实现同时修改播放控制器或资料库 schema。

## 第一版候选发布门槛

进入签名分发和发布候选阶段前，以下项目必须全部完成：

- 一个真实本地 MP3 和 FLAC 可在 Android、Windows、iPhone 和 iPad 被索引、持久化、展示和播放。
- 认证 WebDAV 索引与 Range 播放在 Android、Windows、iPhone 和 iPad 通过。
- 队列切换和保存进度不会造成进度回退。
- 顺序、单曲循环、列表循环、随机播放和队列编辑通过自动化及真机回归。
- 搜索、排序、收藏和播放列表基于真实资料库持久化，不依赖演示数据。
- Android、iPhone/iPad 后台控制与 Windows 系统媒体控制保持同步。
- 目录权限撤销、服务器不可用、凭据错误和音频损坏均有明确恢复状态。
- 10,000 首资料库达到已记录的启动、搜索、滚动和内存指标，数据库升级可恢复。
