# Reverie 开发看板

最后更新：2026-07-15

实时任务状态以 [GitHub Project](https://github.com/users/robeshell/projects/2)
为唯一事实来源；本文件保留产品能力地图、阶段定义和定期快照。规划分为多个
能力域；每个能力域最多只能有一张进行中的卡，修改共享播放或资料库核心时
仍需串行。代码完成后移入**待验证**；只有验收条件和相关平台构建全部通过，
才能移入**已完成**。

## 产品能力地图

平台适配只是其中一条工作流。第一版需要同时推进播放器、资料库、远程来源、
性能稳定性和发布工程；不能以“某个平台能构建”代替播放器产品能力完成。

| 能力域 | 已有基础 | 第一版必须补齐 | 后续增强 |
| --- | --- | --- | --- |
| 平台适配 | Flutter 多端工程；macOS/Android 实播；Android/iOS 后台媒体适配层。 | iPhone/iPad Files 与分屏；Android/iOS 真机；Windows 本地/WebDAV/SMTC；音频焦点与中断恢复。 | 桌面托盘、系统快捷键、拖放、平台安装与自动更新。 |
| 播放器核心 | 播放/暂停/seek/切歌/自动下一首；真实进度源；队列与会话恢复。 | 顺序/单曲循环/列表循环/随机；队列查看、调整、删除、下一首播放；无缝切歌；完整格式矩阵。 | 淡入淡出、ReplayGain/音量标准化、均衡器、倍速和输出设备选择。 |
| 资料库与组织 | SQLite 资料库；本地与 WebDAV 扫描；专辑/歌曲/歌词/封面；原子重扫。 | 搜索、筛选、排序；艺人/专辑/流派入口；多碟专辑；增量扫描；收藏、历史和播放列表。 | 智能播放列表、元数据编辑、重复项与移动文件识别、统计。 |
| 网络与离线 | WebDAV 连接、凭据、目录扫描、Range 播放、seek、LRU 缓存。 | 离线下载管理；网络切换与断线重试；缓存状态、用量和清理 UI；来源失效恢复。 | 多来源合并、冲突处理、可选的其他音乐服务器协议。 |
| 性能与资源 | 元数据在 isolate 解析；SQLite 持久化；基础图片与音频缓存。 | 1,000/10,000 首基准；分页查询和列表虚拟化；启动、扫描、搜索、内存、图片和电量优化。 | 超大库压力测试、后台扫描调度和性能回归基线。 |
| 稳定性与数据安全 | 扫描事务、旧播放事件隔离、缓存并发保护、权限状态模型。 | 可重试错误；离线/凭据过期/损坏音频诊断；数据库迁移、备份恢复和异常退出恢复。 | 日志导出、崩溃诊断和可选的隐私友好遥测。 |
| 产品体验 | 原设计语言；统一 Reverie 组件系统；响应式资料库、专辑页、正在播放页、迷你播放器和移动导航。 | 空状态与首次引导；同步歌词体验；完整键盘操作与焦点导航；触控手势；无障碍、动态字体与中英文。 | 可编辑快捷键、主题定制、动效、桌面紧凑模式和更多视觉细节。 |
| 工程与发布 | 自动化测试、静态检查和各平台 Debug 构建流程。 | CI 构建矩阵；集成/截图回归；版本迁移测试；签名、安装包、隐私说明和开源许可。 | Beta 分发、崩溃符号管理、发布自动化和升级策略。 |

## 当前焦点

SND-204 的原生连续播放、SND-504 的真实资料库入口、SND-601 的收藏与历史、
SND-602 的播放列表编辑、SND-607 的键盘操作、SND-608 的全局设计系统，以及
SND-505 的多碟归组和 SND-603 的歌词时间轴已经完成并进入待验证。
**SND-506 — 增量扫描与变更识别** 的本地与 WebDAV 实现也已进入
待验证；**SND-205 — 音频格式与异常文件矩阵** 的实现也已转入待验证。
SND-304 的离线下载与协议无关 provider 改造已经完成；本轮继续完成了
**SND-305 — 网络变化与来源恢复**、**SND-503 — 生产级错误恢复和诊断**、
**SND-604 — 睡眠定时与常用播放工具** 和 **SND-605 — 首次使用、空状态和权限引导**，
四张卡均已进入待验证：

- 播放器：just_audio 已一次装载完整队列并由原生 currentIndex 推进歌曲；循环模式、
  队列编辑、播放历史和会话状态保持同步，曲间位置用 NativePositionGate 隔离。
  macOS 已用两段真实 FLAC 自然播放验证同会话 0 → 1 切换；Web 明确降级为自动
  连播但不承诺无缝，其他平台等待真机/主机听感验证。
- 资料库：SND-505 已统一本地/WebDAV 的专辑艺人、合辑、同名专辑和 CD/Disc
  归组，并在专辑页按碟号分段；SND-506 已让本地和 WebDAV 重扫按路径、大小和
  修改时间复用未变化元数据与歌词，唯一指纹匹配的移动文件保留歌曲 ID，
  SQLite 只提交真实差量。WebDAV PROPFIND 会请求并解析标准大小、修改时间和
  ETag；当服务器不提供可靠大小/时间时保守地重读元数据。来源页可取消本地或
  WebDAV 扫描，取消和整体扫描失败都不会替换上一版歌曲快照。
- 产品体验：SND-607 已支持全局播放/切歌、搜索和页面快捷键、焦点导航、Esc
  返回/关闭与输入隔离；资料库、搜索、专辑、艺人/流派、收藏/历史和播放列表的
  歌曲行现已在桌面端单击选中、双击或 Enter 播放，触屏端保持单击播放，播放队列
  保持单击切歌。macOS 已补上焦点控件抢占 Space 的真实窗口回归，继续等待完整
  纯键盘路径、系统媒体键和 Windows/iPad 外接键盘人工回归。
- 设计系统：SND-608 已完成兼容式 Reverie V2 迁移并设为产品默认。暖白画布、珊瑚强调色、
  三级文字、边框、浮层和交互状态全部使用语义 token；页面以 16/24/32 dp 响应式边距
  适配紧凑、中等和宽窗口。侧栏、移动导航、弹窗与底部面板使用共享
  `SoundGlassSurface`，长列表和设置卡片使用不重复模糊的轻量透明层；桌面迷你播放器
  改为全窗底部 Dock，进度贴顶贯穿、歌曲信息靠左、播放控制同排居中、工具靠右，移动端
  保留导航栏上方紧凑播放器。macOS/Windows/Linux 始终保留侧栏与桌面 Dock，原生窗口
  最小尺寸为 900×600，不会因窗口变矮而切到手机导航；macOS 使用隐藏标题、透明标题栏
  和全尺寸内容视图，应用背景延伸到红黄绿窗口按钮后方。资料库、专辑、搜索、收藏、
  历史、播放列表、队列、设置、来源、WebDAV 与正在播放均已去除暗色专用文字和边框；
  专辑色正在播放背景保留暖色遮罩，歌词直接融入背景而不再被卡片包裹。设置已重构为
  播放、资料库、操作和关于四组，音乐来源是资料库二级页面，持久化 URI 显示为解码后
  的人类可读路径。完整 271 项测试、静态检查、macOS/Android/iOS 模拟器与 Web
  Debug 构建通过，等待用户按真实内容做视觉验收。
- 歌词：本地与 WebDAV 已统一清洗内嵌 LRC，旧资料库和旧播放会话会在读取时
  自动去除时间戳；同步歌词支持当前行、居中跟随、点击跳转和 ±0.5 秒偏移，
  纯文本歌词按行显示，手机正在播放页新增封面/歌词切换。同步模型已参考 Namida、
  Feishin、Better Lyrics 和 just_audio 的事件模型重构：播放位置是唯一选行依据，
  同时间戳按一个 cue 高亮，seek/换歌取消旧滚动，用户滚动会暂停自动跟随；不再
  根据文案猜测制作人员并改变时间轴。macOS 真实 WebDAV FLAC 已由用户确认声音、
  秒数和歌词一致，继续等待其他格式与平台交互回归。
- 性能：SND-502 已建立内存管线与真实 SQLite 基线，并把 10,000 首同内容重扫从
  约 1.22 秒降到约 131ms；10,000 首合成长列表已加入虚拟化基准，封面按显示尺寸
  解码。SND-201 的播放会话已升级为 v3：队列结构与高频位置 checkpoint 分文件保存，
  普通进度更新不再重写整条队列；会话只保留当前歌曲歌词作为重启恢复兜底，旧 v1/v2
  文件会兼容读取并在下次保存时迁移。Profile 和真机资源采集暂缓，不占用当前功能开发主线。
- 网络与离线：SND-304 已完成第二阶段可用链路。WebDAV 播放临时缓存与用户主动
  离线保存现在使用同一文件但拥有独立语义；固定内容不会被 LRU 或“清理临时缓存”
  删除。专辑页支持整张或逐曲离线、下载进度、取消和失败重试；下载中心会把缓存 URL
  映射回真实歌曲元数据，集中展示下载中、失败和已完成项目，并支持取消、重试、忽略失败
  记录和逐项移除。取消会终止 HTTP 连接并清理临时文件。设置页仍可查看离线/临时缓存
  的文件数和用量，分别安全清理或确认后全部删除。缓存清单可向旧版本兼容恢复，凭据不会
  写入清单，并发播放缓存与离线请求会合并为一次下载。产品层已迁移到协议无关的
  `OfflineDownloadController` 与 `OfflineMediaProvider`：页面不再认识 WebDAV，缓存内容
  使用 provider/resource 复合身份；双 provider 自动化验证了路由隔离和容量聚合。
  播放链路也已建立 `PlaybackMediaProviderRegistry`：`just_audio` 引擎和播放控制器不再
  判断 WebDAV，URI、连接级鉴权、自签名 TLS、精确时序本地缓存和后台缓存均由
  `WebDavPlaybackMediaProvider` 解析。`Track` 与资料库映射不再携带认证 header；离线
  provider 同样从连接规则取凭据。来源类型已改为可扩展标识，并建立
  `SourceProviderRegistry` 统一名称与能力声明；未知协议标识可以穿过 SQLite 和领域层，
  新增协议不再要求修改固定枚举或迁移数据库。重扫已统一由
  `SourceScanProviderRegistry` 路由，本地和 WebDAV
  的参数组装、凭据与取消留在各自适配器；目录选择器也已改为消费通用
  `SourceDirectoryBrowser`，不再直接发 WebDAV 请求。连接观察、探测、浏览入口和删除
  已接入 `SourceConnectionProviderRegistry`；远程来源区由适配器列表驱动，资料库来源
  筛选也根据真实 `SourceKind` 动态生成。契约测试已让第二协议标识贯穿全部共享边界。
- 恢复与诊断：离线、超时、服务器不可用、凭据失效、文件缺失、损坏音频、存储和
  数据库异常已统一映射为可操作的安全提示，原始技术信息只进入本次运行的诊断记录。
  远程播放遇到瞬时故障会按 2/5/10 秒退避重试，并保留队列、当前索引和 seek 位置；
  应用回到前台会再次尝试，凭据或来源问题则引导用户编辑来源而不盲目循环。
  设置页可以查看、复制和清空诊断；全局错误浮层提供重试、编辑来源、定位文件或
  管理存储等恢复入口。
- 首次使用与常用工具：空资料库且没有来源时会显示首次引导，解释本地文件夹和
  WebDAV 的用途，并可直接进入添加来源流程；“稍后”只跳过当前运行。设置页新增
  15/30/45/60 分钟睡眠定时、播完当前歌曲停止和取消入口，定时器使用绝对截止时间
  校准后台停留，播完当前歌曲以实际下一首开始事件为停止边界。
- 本轮验证：完整 282 项自动化测试、静态分析和 diff 检查通过；macOS Debug、
  Android Debug APK、iPhone/iPad 模拟器 Debug 与 Web Debug 构建通过。移动端后台行为、真实网络
  切换和通知/锁屏交互仍等待真机验收，Windows 构建仍需 Windows 主机。

Windows 媒体控制受当前主机限制，SND-401/SND-403 保持阻塞，不妨碍
播放器和资料库功能继续开发。
