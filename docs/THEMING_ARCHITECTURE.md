# Reverie 外观架构

Reverie 的外观由三个互相独立的维度组合，而不是由一个主题枚举接管所有界面：

| 维度 | 负责 | 不负责 |
| --- | --- | --- |
| Skin | 画布、表面层级、文字、玻璃材质、阴影和背景动效 | 页面信息结构、品牌强调色 |
| Accent | 按钮、图标、焦点、选中和当前播放状态 | 背景与文字层级 |
| Player style | 正在播放页的布局比例和移动端默认视图 | 全局颜色和材质 |

## 数据流

`ThemePreferences` 持久化三个稳定 ID。`SoundApp` 是运行时唯一状态源，负责加载、
即时更新、处理启动阶段的写入竞争并串行保存。`AppShell` 只负责把播放器样式和更新
回调传到设置页与正在播放页。

皮肤由 `SoundSkinPreset` 定义，并通过 `SoundTheme.forSkin()` 编译为完整的 Flutter
`ThemeData`：

- `ColorScheme` 提供画布与表面语义；
- `SoundGlassTheme` 提供玻璃、边框和三级文字语义；
- `SoundSkinEffects` 提供背景运动、光晕、遮罩与阴影强度；
- `AccentPreset` 注入强调色及 hover/pressed 状态。

组件只消费这些语义 token，不应判断皮肤 ID。新增皮肤原则上只需新增一个 preset，
不应复制页面或添加 `if (skin.id == ...)`。

## 响应式边界

设备适配与皮肤保持独立：窗口宽度和平台决定 compact / medium / wide 信息架构；
折叠屏额外读取 `displayFeatures` 保护中轴区域。皮肤在所有尺寸共享同一组语义，
播放器样式再按当前窗口映射为双栏比例或移动端初始视图。

## 兼容原则

- `default` 是冻结的 Reverie 基准皮肤；扩展皮肤不能改变它的 token。
- 珊瑚色始终是默认品牌强调色，并排在强调色列表首位。
- 缺失或未知的持久化字段必须回退到默认皮肤、珊瑚色和经典双栏。
- 系统“减少动态效果”优先于所有皮肤动效参数。

## 阶段规划

1. 稳定默认皮肤及跨平台基础 token。
2. 将播放器结构拆为可持久化、响应式的独立样式。
3. 让官方皮肤拥有独立材质、阴影与背景动效特征。
4. 增加轻量自定义主题色：在安全的饱和度与明度范围内选择任意色相，自动生成
   hover、pressed 和可读前景色，并与官方主题色使用同一持久化链路。

音乐桌面和完整自定义皮肤编辑器不在当前产品范围内；如无新的产品决策，不为它们
预留页面分支或增加运行时复杂度。
